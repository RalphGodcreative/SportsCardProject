# Implementation Plan: Player Recommendation Table

## Summary

A new `recommendations` table storing per-sport player write-ups. `sport` is a fixed enum (`BASEBALL`, `BASKETBALL`, `FOOTBALL`). `reason` is a short note targeting 70-90 characters, backed by a `VARCHAR(120)` column for headroom. Entries can be added **manually** by Ralph (`author = ADMIN`) or **generated by AI** (`author = AI`, via Gemini, reusing the existing `GeminiService`). Not tied to a specific `Card` row — recommendations are about a player + sport in general, independent of whether that player is already in the collection. Global/shared table, not scoped per-user.

History is kept: regenerating/re-evaluating a player creates a **new row** rather than overwriting the old one, so the outlook over time is visible (matches the mindset already used for the Yahoo Auctions sold-price crawler — see `[[landing-and-moat-direction]]`).

## 3. `Recommendation.java` — New Entity

```java
package RGcards.SportsCardProject.entity;

import RGcards.SportsCardProject.enums.RecommendationAuthor;
import RGcards.SportsCardProject.enums.RecommendationSport;
import jakarta.persistence.*;
import lombok.AllArgsConstructor;
import lombok.Data;
import lombok.NoArgsConstructor;

import java.time.LocalDateTime;

@Data
@NoArgsConstructor
@AllArgsConstructor
@Entity
@Table(name = "recommendations")
public class Recommendation {

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private int id;

    private String player;

    @Enumerated(EnumType.STRING)
    private RecommendationSport sport;

    @Column(length = 120)
    private String reason;

    @Enumerated(EnumType.STRING)
    private RecommendationAuthor author;

    private LocalDateTime createdAt;
}
```

## 6. `RecommendationService.java` — New Service (Manual CRUD)

```java
package RGcards.SportsCardProject.service;

import RGcards.SportsCardProject.entity.Recommendation;
import RGcards.SportsCardProject.enums.RecommendationAuthor;
import RGcards.SportsCardProject.repository.RecommendationRepository;
import lombok.RequiredArgsConstructor;
import org.springframework.stereotype.Service;

import java.time.LocalDateTime;
import java.util.List;

@Service
@RequiredArgsConstructor
public class RecommendationService {

    private final RecommendationRepository recommendationRepository;

    public List<Recommendation> findAll() {
        return recommendationRepository.findAllByOrderByCreatedAtDesc();
    }

    public Recommendation saveManual(Recommendation recommendation) {
        recommendation.setAuthor(RecommendationAuthor.ADMIN);
        recommendation.setCreatedAt(LocalDateTime.now());
        return recommendationRepository.save(recommendation);
    }
}
```

## 7. `RecommendationAiService.java` — New Service (AI Generation)

Mirrors the existing `CardAiService` pattern, but prompts about a player in general rather than a specific graded card.

```java
package RGcards.SportsCardProject.service;

import RGcards.SportsCardProject.entity.Recommendation;
import RGcards.SportsCardProject.enums.RecommendationAuthor;
import RGcards.SportsCardProject.enums.RecommendationSport;
import RGcards.SportsCardProject.repository.RecommendationRepository;
import com.fasterxml.jackson.core.JsonProcessingException;
import lombok.RequiredArgsConstructor;
import org.springframework.stereotype.Service;

import java.time.LocalDateTime;

@Service
@RequiredArgsConstructor
public class RecommendationAiService {

    private final GeminiService geminiService;
    private final RecommendationRepository recommendationRepository;

    public Recommendation generate(RecommendationSport sport, String player) throws JsonProcessingException {
        String prompt = String.format(
            "You are a sports card investment analyst. Research this player's current stats, recent " +
            "performance, and market trends for their trading cards.\n" +
            "Sport: %s\nPlayer: %s\n\n" +
            "Reply with a short reason why this player is worth watching for card investment, " +
            "70-90 characters total, no more. No preamble, no labels, just the reason.",
            sport, player
        );

        String raw = geminiService.generateContent(prompt, "gemini-2.5-flash", true);

        Recommendation recommendation = new Recommendation();
        recommendation.setSport(sport);
        recommendation.setPlayer(player);
        recommendation.setReason(truncate(raw, 120));
        recommendation.setAuthor(RecommendationAuthor.AI);
        recommendation.setCreatedAt(LocalDateTime.now());

        return recommendationRepository.save(recommendation);
    }

    private String truncate(String text, int maxLength) {
        if (text == null || text.length() <= maxLength) return text;
        return text.substring(0, maxLength);
    }
}
```

The prompt targets 70-90 characters, but LLM output length isn't guaranteed — `truncate` is a hard backstop so a longer-than-expected response can't violate the `VARCHAR(120)` column and blow up the insert.

## 8. `RecommendationController.java` — New REST Controller

Read-focused REST API showing recommendations from **both** authors (`ADMIN` and `AI`) together — mirrors `CardRestController`'s style (manual `ObjectMapper` serialization, `@RestController`).

```java
package RGcards.SportsCardProject.controller;

import RGcards.SportsCardProject.entity.Recommendation;
import RGcards.SportsCardProject.service.RecommendationService;
import com.fasterxml.jackson.core.JsonProcessingException;
import com.fasterxml.jackson.databind.ObjectMapper;
import lombok.RequiredArgsConstructor;
import org.springframework.web.bind.annotation.*;

import java.util.List;

@RestController
@RequestMapping("/recommendation")
@RequiredArgsConstructor
public class RecommendationController {

    private final RecommendationService recommendationService;

    // GET /recommendation -> all recommendations (admin + AI), newest first
    @GetMapping
    public String findRecommendations() throws JsonProcessingException {
        List<Recommendation> recommendations = recommendationService.findAll();
        ObjectMapper om = new ObjectMapper();
        return om.writeValueAsString(recommendations);
    }
}
```

Since the table is global (no `user_id`) and this is a read-only endpoint, it does not need `@AuthenticationPrincipal User currentUser` the way `CardRestController` does. Whether the route sits behind login or is `permitAll` is a `SecurityConfig` decision — flag if you want it public (e.g. for the future landing-page moat content per `[[landing-and-moat-direction]]`) vs. behind login like the rest of the app.

## 9. `SecurityConfig.java` — Admin-Only Route

**Blocker to resolve first:** roles are defined in `docs/user-roles.md` but nothing in the app ever grants `ROLE_ADMIN` — both signup paths (`AuthController.java:58` and `CustomOAuth2UserService.java:31`) hardcode `user.setRole("ROLE_USER")`, and `SecurityConfig` currently only distinguishes `permitAll` vs `authenticated()`, never checking role. Before this page can be "admin-only" for real, promote your own account manually:

```sql
UPDATE users SET role = 'ROLE_ADMIN' WHERE id = 1;
```

Then add a role check to the filter chain (`User.getRole()` already stores the full `"ROLE_ADMIN"` string via `getAuthorities()`, so use `hasAuthority`, not `hasRole` which would expect the prefix stripped):

```java
.authorizeHttpRequests(auth -> auth
    .requestMatchers(
        "/", "/random", "/randomize",
        "/login", "/register",
        "/assets/**", "/css/**", "/js/**", "/images/**", "/img/**"
    ).permitAll()
    .requestMatchers("/admin/**").hasAuthority("ROLE_ADMIN")
    .anyRequest().authenticated()
)
```

## 10. `RecommendationAdminController.java` — New Controller (Thymeleaf)

Admin-only page to write a recommendation manually and to trigger AI generation for a player/sport. Mirrors `CardController`'s Thymeleaf style, mapped under `/admin/recommendations` so the `hasAuthority("ROLE_ADMIN")` rule above covers it.

```java
package RGcards.SportsCardProject.controller;

import RGcards.SportsCardProject.entity.Recommendation;
import RGcards.SportsCardProject.enums.RecommendationSport;
import RGcards.SportsCardProject.service.RecommendationAiService;
import RGcards.SportsCardProject.service.RecommendationService;
import com.fasterxml.jackson.core.JsonProcessingException;
import lombok.RequiredArgsConstructor;
import org.springframework.stereotype.Controller;
import org.springframework.ui.Model;
import org.springframework.web.bind.annotation.*;

@Controller
@RequestMapping("/admin/recommendations")
@RequiredArgsConstructor
public class RecommendationAdminController {

    private final RecommendationService recommendationService;
    private final RecommendationAiService recommendationAiService;

    @GetMapping
    public String view(Model model) {
        model.addAttribute("recommendations", recommendationService.findAll());
        model.addAttribute("sports", RecommendationSport.values());
        return "recommendationAdmin";
    }

    @PostMapping
    public String saveManual(@ModelAttribute("player") String player,
                              @ModelAttribute("sport") RecommendationSport sport,
                              @ModelAttribute("reason") String reason) {
        Recommendation recommendation = new Recommendation();
        recommendation.setPlayer(player);
        recommendation.setSport(sport);
        recommendation.setReason(reason);
        recommendationService.saveManual(recommendation);
        return "redirect:/admin/recommendations";
    }

    @PostMapping("/generate")
    public String generate(@ModelAttribute("player") String player,
                            @ModelAttribute("sport") RecommendationSport sport) throws JsonProcessingException {
        recommendationAiService.generate(sport, player);
        return "redirect:/admin/recommendations";
    }
}
```
